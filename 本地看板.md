## 四、预警规则配置

| 规则ID | 名称 | 触发条件 | 严重度 | 检查窗口 |
|---|---|---|---|---|
| rule-negative-sentiment | 负面舆情激增 | 负面情感占比 > 60% | 警告 | 24小时 |
| rule-new-risk | 风险事件新增 | 24h内新增未解决风险事件 | 严重 | 24小时 |
| rule-reputation-drop | 口碑分骤降 | 近7天情感走势下降 > 15% | 警告 | 7天 |
| rule-communication-spike | 传播力异常波动 | 近7天声量增减 > 30% | 提醒 | 7天 |


访问：http://localhost:3000

### 方式二：静态模式（生产构建）

```bash

npm run build
npx serve out -l 3456
```

访问：http://localhost:3456

- `out/index.html` → 首页
- `out/ip-management/index.html` → IP库

- `out/compare/index.html` → 对比分析
- `out/cooperation/index.html` → 合作评估
- `out/alerts/index.html` → 预警中心

## 七、环境配置

编辑 `.env.local`：

```bash
# Mock 模式（默认，无后端也可运行）
NEXT_PUBLIC_DATA_MODE=mock

# REST 模式（需后端 API 支持）
NEXT_PUBLIC_DATA_MODE=rest
NEXT_PUBLIC_API_BASE_URL=http://localhost:8080/api
```

配置变更后需重新构建：

```bash
npm run build
```
